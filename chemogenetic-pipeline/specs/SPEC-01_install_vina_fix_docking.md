**Priority:** CRITICAL
**Module:** 4 (Molecular Docking)
**Depends on:** Nothing
**Blocks:** SPEC-05, SPEC-07, SPEC-08

## Problem Statement

AutoDock Vina is not installed. Every call to `dock_compound()` in `src/module4/docking_engine.py` catches the `ImportError` from `from vina import Vina` and falls back to `_estimate_docking()`, which computes:

```python
affinity = -5.0 - (n_atoms / 10.0) + np.random.normal(0, 0.5)
```

This is a heuristic based on atom count + random noise. It has zero relationship to actual binding. Every row in `data/results/docking_results.csv` has `estimated=True`. No docked pose files exist (`data/docking/poses/` directory is empty or missing).

## Current State

- **File:** `src/module4/docking_engine.py`
- **Fallback function:** `_estimate_docking(ligand_pdbqt)` — counts HETATM/ATOM lines, applies formula, clips to [-12, -3] kcal/mol
- **Receptor PDBQT exists:** `data/docking/receptor/8E9W_clean_fixed.pdbqt` (verified)
- **Ligand PDBQTs exist:** generated by `src/module4/ligand_prep.py` using RDKit + Meeko (verified working)
- **Binding site hardcoded in** `src/module4/receptor_prep.py`:
  ```python
  BINDING_SITE = {
      "center_x": -10.0, "center_y": 5.0, "center_z": -15.0,
      "size_x": 28.0, "size_y": 28.0, "size_z": 28.0,
  }
  ```
- **Vina package:** NOT INSTALLED (confirmed via `python3 -c "from vina import Vina"` → ImportError)
- **Poses directory:** `data/docking/poses/` does not exist

### Step 5: No code changes needed in docking_engine.py

The existing code in `dock_compound()` already has the real Vina path:
```python
try:
    from vina import Vina
    v = Vina(sf_name='vina')
    v.set_receptor(receptor_pdbqt)
    v.set_ligand_from_file(ligand_pdbqt)
    v.compute_vina_maps(center=center, box_size=box_size)
    v.dock(exhaustiveness=exhaustiveness, n_poses=n_poses)
    ...
except ImportError:
    return _estimate_docking(ligand_pdbqt)  # THIS is the problem
```

Once Vina is installed, the `ImportError` will no longer be raised, and real docking will execute.
